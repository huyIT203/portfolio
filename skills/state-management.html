State Management

Managing Application State in Flutter

Level:
Proficient

Overview

State Management is an important part of Flutter app development. Proper state management helps applications run efficiently and are easy to maintain.

I have experience working with different state management solutions, from basic Provider to complex solutions like BLoC and Riverpod.

Solutions Used

Provider

  • ChangeNotifier
  • Consumer & Selector
  • MultiProvider
  • ProxyProvider

BLoC / Cubit

  • Event-driven architecture
  • BlocBuilder & BlocListener
  • BlocConsumer
  • Cubit for simple states

GetX

  • Reactive state
  • GetBuilder
  • Dependency injection
  • Route management

Riverpod

  • Providers types
  • ConsumerWidget
  • Ref & watch
  • State notifier

When to Use What?

Provider

Small to medium apps, need simplicity and easy to learn

BLoC

Large apps, need high testability and clear separation of concerns

GetX

Need all-in-one (state, route, dependency), rapid prototyping

Riverpod

Advanced Provider, need compile-time safety and flexibility

BLoC Example

counter_bloc.dart Dart 
// Events
abstract class CounterEvent {}
class IncrementEvent extends CounterEvent {}
class DecrementEvent extends CounterEvent {}

// State
class CounterState {
  final int count;
  CounterState(this.count);
}

// BLoC
class CounterBloc extends Bloc<CounterEvent, CounterState> {
  CounterBloc() : super(CounterState(0)) {
    on<IncrementEvent>((event, emit) {
      emit(CounterState(state.count + 1));
    });
    
    on<DecrementEvent>((event, emit) {
      emit(CounterState(state.count - 1));
    });
  }
}

// Usage in Widget
BlocBuilder<CounterBloc, CounterState>(
  builder: (context, state) {
    return Text('Count: ${state.count}');
  },
)
