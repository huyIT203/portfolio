Clean Architecture

Software Design Principles & Patterns

Level:
Intermediate

Overview

Clean Architecture is a software architecture that helps separate concerns and create maintainable, testable and scalable code. This architecture was proposed by Robert C. Martin.

In Flutter projects, I apply Clean Architecture to organize code by layers, making development and maintenance easier.

Architecture Layers

Domain Layer

Entities, Use Cases, Repository Interfaces

Data Layer

Repository Impl, Data Sources, Models

Presentation Layer

UI, BLoC/Provider, Widgets

SOLID Principles

S

Single Responsibility

Each class should have only one reason to change

O

Open/Closed

Open for extension, closed for modification

L

Liskov Substitution

Subtypes must be substitutable for their base types

I

Interface Segregation

Many small interfaces are better than one large interface

D

Dependency Inversion

Depend on abstractions, not concretions
